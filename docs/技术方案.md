# BehaviorRecord 数据表设计方案

## 需求背景

当前 ActionSchema 存在以下问题：
1. **缺少参数记录**：ParameterAgent 选择的 `parameters`（如选择的食物名称、价格等）没有保存到数据库
2. **来源单一**：只能记录 Agent 产生的行为，无法记录用户操作（如给金币）、系统事件等
3. **元数据不足**：无法记录行为的附加信息（如持续时间等）
4. **语义不清晰**：表名和字段命名不够准确，难以理解其用途

## 数据表设计

### 接口定义

```typescript
export interface IBehaviorRecord extends Document {
  type: string;                              // 行为/事件类型
  description: string;                       // 行为描述
  timestamp: Date;                           // 发生时间

  // 行为来源
  trigger: 'agent' | 'user' | 'system';      // 触发来源

  // 行为参数（可选）
  parameters?: BehaviorParameter[];          // Agent 选择的行为参数，如选择的食物

  // 行为持续时间（可选）
  duration_minutes?: number;                 // 持续时间（分钟）
}
```

### Schema 定义

```typescript
const BehaviorRecordSchema = new Schema<IBehaviorRecord>({
  type: { type: String, required: true, index: true },
  description: { type: String, required: true },
  timestamp: { type: Date, required: true, default: Date.now, index: true },

  trigger: {
    type: String,
    enum: ['agent', 'user', 'system'],
    required: true,
    default: 'agent',
    index: true,
  },

  parameters: [
    {
      value: { type: Schema.Types.Mixed, required: true },
      description: { type: String, required: true },
      extra: { type: Schema.Types.Mixed, default: {} },
    },
  ],

  duration_minutes: { type: Number },
});

// 复合索引
BehaviorRecordSchema.index({ timestamp: -1 });
BehaviorRecordSchema.index({ trigger: 1, timestamp: -1 });
BehaviorRecordSchema.index({ type: 1, timestamp: -1 });
```

### 类型定义

```typescript
/**
 * 行为参数
 * 用于存储 ParameterAgent 选择的具体参数
 * 例如：Agent 选择了 Eat_Item 行为，ParameterAgent 会进一步选择吃什么（苹果、面包等）
 */
export interface BehaviorParameter {
  /** 参数值，如："苹果"、"面包" */
  value: any;
  /** 参数描述，如："苹果可以恢复15点体力" */
  description: string;
  /** 额外信息，如：{ stamina: 15, price: 5, quantity: 2 } */
  extra?: Record<string, any>;
}
```

## 字段说明

| 字段               | 类型                | 必填 | 默认值   | 说明                                                       |
| ------------------ | ------------------- | ---- | -------- | ---------------------------------------------------------- |
| `type`             | String              | 是   | -        | 行为/事件类型，如 `eat_item`、`buy_item`、`receive_money`  |
| `description`      | String              | 是   | -        | 行为描述，如 "肚子饿了，吃个苹果"                          |
| `timestamp`        | Date                | 是   | Date.now | 行为发生时间                                               |
| `trigger`          | String              | 是   | 'agent'  | 触发来源：`agent`（LLM）、`user`（用户）、`system`（系统） |
| `parameters`       | BehaviorParameter[] | 否   | -        | Agent 选择的行为参数，如选择的食物、购买的物品             |
| `duration_minutes` | Number              | 否   | -        | 行为持续时间（分钟）                                       |

### BehaviorParameter 字段说明

| 字段          | 类型                | 必填 | 说明                                               |
| ------------- | ------------------- | ---- | -------------------------------------------------- |
| `value`       | Any                 | 是   | 参数值，如 "苹果"、"面包"、"5"（购买数量）等       |
| `description` | String              | 是   | 参数描述，如 "苹果可以恢复15点体力"                |
| `extra`       | Record<string, any> | 否   | 额外信息，如 { stamina: 15, price: 5, quantity: 2 } |

## 使用示例

### 场景 1：Agent 选择吃苹果

```typescript
{
  type: 'eat_item',
  description: '肚子饿了，吃个苹果',
  timestamp: new Date('2025-12-28 12:00:00'),
  trigger: 'agent',
  parameters: [{
    value: '苹果',
    description: '苹果可以恢复15点体力',
    extra: { stamina: 15, category: 'food', quantity: 2 }
  }],
  duration_minutes: 10
}
```

## 设计优势

1. **语义清晰**：表名 `BehaviorRecord` 和字段命名更准确，易于理解
2. **参数结构清晰**：`parameters` 字段准确反映 ParameterAgent 的选择结果
3. **灵活扩展**：`extra` 字段支持任意附加信息，适应不同行为的需求
4. **完整索引**：为常用查询字段建立索引，提升查询性能
5. **来源明确**：`trigger` 字段清晰区分行为来源，便于数据分析

## 数据库索引

```typescript
// 按时间倒序查询最近行为
BehaviorRecordSchema.index({ timestamp: -1 });

// 按触发来源和时间查询
BehaviorRecordSchema.index({ trigger: 1, timestamp: -1 });

// 按行为类型和时间查询
BehaviorRecordSchema.index({ type: 1, timestamp: -1 });
```

## 注意事项

1. **type 字段**：使用字符串类型，支持任意行为类型，不仅限于 ActionId 枚举
2. **trigger 字段**：明确区分行为来源，便于后续数据分析和审计
3. **parameters 字段**：存储 ParameterAgent 选择的具体参数，如选择的食物、购买的物品等
4. **parameters.value**：参数值可以是任意类型（字符串、数字、对象等）
5. **parameters.extra**：用于存储参数的额外信息，如价格、体力恢复值等
6. **duration_minutes 字段**：记录行为持续时间，可用于分析行为模式和时间分配
