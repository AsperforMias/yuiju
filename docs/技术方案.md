# 悠酱 (Yuiju) 状态存储与记忆系统技术方案

## 1. 核心状态存储 (Redis)

为了满足 AI 角色高频状态更新（心跳机制）与持久化的需求，采用 **Redis** 作为运行时状态（Runtime State）的存储介质。

### 1.1 架构设计
采用 **Repository 模式** 将业务逻辑与底层存储解耦。
- **Entity**: 保持现有的 `CharactorState` 和 `WorldState` 类定义，作为数据传输对象。
- **Repository**: 负责 Redis 的读写操作，对外暴露 `save`, `load`, `update` 等方法。

### 1.2 数据结构设计 (Key Schema)

所有 Key 使用 `yuiju:` 前缀，防止命名冲突。

#### A. 世界状态 (WorldState)
- **Key**: `yuiju:world:state`
- **Type**: `Hash`
- **Fields**:
  - `time`: timestamp 时间戳，单位毫秒
- **说明**: 存储世界流逝的时间，由 TimeSystem 定期更新。

#### B. 角色状态 (CharactorState)
- **Key**: `yuiju:charactor:state`
- **Type**: `Hash`
- **Fields**:
  - `stamina`: number (0-100)
  - `money`: number
  - `action`: string (ActionId)
  - `location`: JSON string (e.g., `{"major": "Home"}`)
- **说明**: 存储角色的核心属性。`location` 因包含嵌套结构，序列化为 JSON 字符串存储。

#### C. 每日行为记录 (Daily Actions)
- **Key**: `yuiju:charactor:daily_actions`
- **Type**: `Set`
- **Values**: ActionId (e.g., "sleep", "eat_lunch")
- **TTL**: **自动过期**。
  - 策略：每次写入时，检查 TTL。如果未设置，则设置为 **次日 00:00:00** 过期。
  - 优势：利用 Redis TTL 自动处理“每日重置”逻辑，无需编写定时任务清理代码。

### 1.3 交互流程
1. **初始化**: 系统启动时，Repository 从 Redis 加载状态。若 Redis 为空，使用默认初始值并回写 Redis。
2. **更新**: 
   - 属性变更（如扣除体力）直接调用 `Redis.HINCRBY` 保证原子性。
   - 状态变更（如切换动作）调用 `Redis.HSET`。
3. **读取**: 实时从 Redis 读取，确保多进程/多实例下的状态一致性。

## 2. 短期记忆系统改造 (MongoDB)

**目标**: 移除内存中的 `ShortActionMemory` 缓存队列，实现“单一数据源”，直接利用 MongoDB 存储的历史记录作为短期记忆。

### 2.1 现状问题
- `ShortActionMemory` 维护了一个内存数组 `buffer`。
- 同时又在 `push` 时异步写入 MongoDB。
- 存在数据冗余，且重启后内存数据丢失，与数据库不一致。

### 2.2 改造方案

#### A. Utils 层 (`packages/utils`)
在 `src/db/schema/action.schema.ts` 中扩展数据库操作能力：
1. **新增查询方法**: `getRecentActions(limit: number): Promise<IActionSchema[]>`
   - 逻辑: `ActionModel.find().sort({ create_time: -1 }).limit(limit)`
   - 作用: 获取最近发生的 N 次行为及其原因。

#### B. World 层 (`packages/world`)
1. **移除**: 直接删除 `src/memory/short-action.ts` 文件。
2. **重构调用方**: 
   - 修改 `src/engine/tick.ts` 及其他引用处。
   - 直接调用 `getRecentActions(limit)` 获取数据。
   - 将同步调用改为异步 `await` 调用。

### 2.3 影响范围
- **Prompt 构建**: 构建 Prompt 时获取“最近发生的事”需要变为异步。
- **决策系统**: 任何依赖“上一次行为”的逻辑都将变为异步查询。
