### 📂 packages

#### 1. **`message` (交互层)**

- **定位**：负责对外交互与消息处理，目前主要实现了一个 QQ 机器人客户端。
- **核心功能**：
  - 基于 `node-napcat-ts` 监听 QQ 消息（目前主要是私聊）。
  - 集成 `mem0ai` 和 LLM 管理器 (`llm/manager`) 进行对话生成。
  - 负责消息的接收、持久化存储 (`saveQQMessage`) 及回复。
  - 包含终端交互模式 (`src/terminal.ts`) 用于调试。

#### 2. **`world` (核心引擎层)**

- **定位**：Agent 的自主生活与模拟引擎，负责驱动角色的状态变化与自主行为。
- **核心功能**：
  - **实时循环 (`engine/runner`)**：运行 Tick 机制，驱动世界时间流逝。
  - **行为决策**：角色的行为由 LLM 进行自主决策。
  - **状态管理 (`state`)**：维护角色状态（如体力、心情等）及世界状态。
  - **行动系统 (`action`)**：定义具体的行为逻辑（如在家休息、去学校等）。

#### 3. **`source` (数据与设定层)**

- **定位**：存放项目的静态资源、设定数据及训练语料。
- **核心功能**：
  - **Prompt (`prompt`)**：包含角色卡 (`character-card`)、世界观 (`world-view`) 等核心 Prompt 定义。

#### 4. **`utils` (基础工具层)**

- **定位**：提供跨包通用的基础设施与工具函数。
- **核心功能**：
  - **数据库 (`db`)**：统一管理数据库连接 (`connectDB`) 及 Schema 定义（如 `qqMessage`, `action`）。
  - **通用工具**：时间处理、日志封装等公共辅助函数。

---

**调用关系简述**：
`message` 和 `world` 均依赖 `utils` 获取底层支持（如 DB），并可能引用 `source` 中的设定数据。`world` 独立运行模拟逻辑，而 `message` 负责处理外部的用户输入。
