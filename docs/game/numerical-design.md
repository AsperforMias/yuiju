# 数值设计

## 设计目标

- 让 LLM 在“恢复体力 / 赚钱 / 消耗金钱”之间做出稳定、可预测的权衡
- 让不同地点的消费与收益保持同一套基础汇率，避免某个系统碾压其它系统
- 保持规则足够简单，便于后续扩展更多地点与商品

## 核心资源

- 体力（stamina）：范围 0-100（状态层会做 clamp）
- 金钱（money）：非负整数
- 时间：行为以分钟为单位推进世界时间

## 基础汇率（锚点）

### 金钱与体力的等价关系

- 基础汇率：1 点体力 ≈ 10 元
- 该汇率由两条规则共同锚定：
  - 体力恢复：价格 ÷ 10 = 体力恢复
  - 打工：每小时 +200 元，同时消耗 -10 体力
