# Web 页面 PRD

## 1. 背景

“悠酱”是一个由 LLM 驱动的虚拟生活模拟系统：世界引擎持续推进时间与行为，角色状态以 Redis 为准，行为记录落到 MongoDB，同时具备对话能力与长期记忆服务。

当前缺少一个面向普通用户的 C 端 Web 页面，用于：

- 让用户直观看到悠酱现在的状态与所在地点
- 让用户回溯悠酱最近做过的事（行为时间线）
- 让用户在网页里直接和悠酱对话
- 提供少量“照顾/调整”入口（轻管理，不做复杂后台）

## 2. 产品定位

- 产品类型：面向用户的 C 端陪伴型 Web 应用（PC 端优先设计，适配移动端）
- 核心价值：
  - 看见：状态与位置可视化
  - 理解：行为时间线回溯
  - 互动：网页聊天
  - 轻照顾：少量管理操作（如发放零花钱）

## 3. 目标（Goals）

- G1：用户打开页面 10 秒内理解“悠酱在哪、在做什么、状态如何”
- G2：用户可按时间线查看行为记录，并在详情中看到数据库原始字段
- G3：用户可设置自己的 user_name，并与悠酱进行连续对话
- G4：地图具备南北方向感，能体现地点的相对地理关系，并高亮当前地点

## 4. 非目标（Non-goals）

- 不做账号体系/登录鉴权（MVP 单实例）
- 不做复杂地图（真实比例尺、路径规划、室内级别导航）
- 不做暗色模式
- 不做“行为解释/总结”的二次推理（Behavior 详情只展示数据库已有字段）

## 5. 用户与使用场景

### 5.1 目标用户

- 普通用户（C 端）：想陪伴悠酱、了解她的生活轨迹、与她聊天

### 5.2 使用场景

- 随时打开手机看看悠酱在干嘛（状态卡片 + 地图定位）
- 翻一翻悠酱今天做了什么（行为时间线）
- 给悠酱发消息聊天（用户可自定义昵称）
- 进行轻量照顾（如发零花钱）

## 6. 信息架构（IA）

PC 端优先，支持将“总览 + 地图 + 对话”合并为一个首页；其他能力作为独立页面。移动端适配为折叠菜单或底部 Tab（可选）。

- 首页（Home）：状态 + 地图 + 对话（同页）
- 动态（Activity）：行为时间线 + 轻管理入口
- 设置（Settings）：user_name

### 6.1 UI 风格与视觉规范

本项目为日系二次元题材，UI 采用“日系简约风”，强调轻盈、留白、圆角与低对比度的舒适阅读体验。

#### 6.1.1 设计关键词

- 日系简约、轻量、克制
- 大留白、低对比、轻阴影
- 圆角卡片、线性图标、纯色块为主（不使用渐变）

#### 6.1.2 色彩规范（主/辅色）

- 主颜色（Primary）：水蓝色 #91c4ee（主按钮、强调信息、链接）
- 辅颜色（Secondary）：淡紫色 #af7ac5（次级强调、选中态、标签）
- 辅颜色（Accent）：暖调浅杏色 #fae3be（提示/徽标/柔和背景块）

建议配套的中性色（用于统一视觉与可读性）：

- 背景色：#f7fbff（大面积底色，避免纯白刺眼）
- 卡片底：#ffffff
- 边框/分割线：#d9e6f5
- 主文字：#2b2f36
- 次文字：#6b7480

#### 6.1.3 组件风格约束

- 卡片：圆角（建议 12px），边框细线 + 轻阴影（避免厚重投影）
- 按钮：
  - 主按钮使用主色 #91c4ee
  - “聊天入口”等次级强调可使用淡紫 #af7ac5（纯色块，不使用渐变）
  - 危险操作尽量不出现在 C 端主路径
- 输入框：圆角 + 轻边框，高亮态使用主色描边
- 标签/徽标：
  - 使用淡紫/浅杏作为背景底色
  - 文本保持克制，避免高饱和色块
- 对话气泡：用户/悠酱使用两种低对比底色区分（不使用强对比黑白）
- 行为时间线：以“卡片 + 细竖线/时间点”呈现，信息密度可控

#### 6.1.4 页面布局（与首页布局一致）

- PC：内容区以“状态 + 地图”为主；聊天以“手机抽屉（侧拉 Drawer）”形式呼出，保持主视图沉浸
- 移动端：主视图默认展示状态与地图；聊天作为底部/侧边抽屉打开，符合“手机里聊天”的世界观设定

## 7. 功能需求（MVP）
### 7.0 首页（Home）整体布局

#### 7.0.1 PC 端布局（同页）

首页默认以“状态 + 地图”为主，聊天不占用常驻布局，通过抽屉呼出：

- 左侧：角色状态 + 世界状态（含刷新按钮、一句话摘要、计划、今日已执行、背包摘要）
- 右侧：地图（节点 + 道路连线，当前地点高亮）
- 顶部/右上：聊天入口按钮（淡紫），点击后从右侧拉出聊天抽屉

#### 7.0.2 移动端适配

移动端以“状态 + 地图”为主，聊天以抽屉形式打开：

- 顶部固定：状态摘要 + 刷新按钮
- 中部：地图（可折叠为“当前地点卡片”）
- 底部：聊天入口（按钮），点击打开聊天抽屉（不常驻占屏）


### 7.1 总览（Dashboard）


- 角色状态（Character）
  - 当前行为：action
  - 当前地点：location（大场景）
  - 体力：stamina
  - 金钱：money
  - 今日已执行的行为：dailyActionsDoneToday（卡片内以标签列表展示）
  - 计划：longTermPlan / shortTermPlan（如有）
  - 背包摘要：inventory（摘要 + 可展开查看全部）
- 世界状态（World）
  - 世界时间：time

#### 7.1.2 交互

- 提供刷新按钮（不轮询状态）
- 顶部提供一句话摘要（例：“悠酱现在在【家】，正在【发呆】”）

### 7.2 地图（Map）

#### 7.2.1 地图形式

采用“示意地图”（非比例尺）来表达地理关系与方向感：

- 地点以节点/卡片形式呈现，并通过空间位置体现方向感
- 道路使用直线连接地点（纯色实线）
- 高亮悠酱当前所在地点（淡紫底色）
- 支持“放大地图”占满当前页面（Overlay），可点击遮罩或按 Esc 退出

#### 7.2.2 地点布局（默认）

为保证南北方向感，MVP 固定布局如下（后续可配置）：

- 北：学校
- 中：家
- 南：商店

#### 7.2.3 地点信息

- 点击地点节点，展示地点简介
- 地点简介来源：世界观文本（第一版不要求结构化地图资源）
- 地点弹窗可展示：地点简介 + 可用行为（标签展示，UI 友好，非必须执行）

### 7.3 动态（Activity）

#### 7.3.1 行为时间线列表

- 默认：展示“今天”的行为记录，按时间倒序
- 列表项字段：
  - timestamp
  - behavior
  - description
  - trigger（agent / user / system）
  - duration_minutes（如有）

#### 7.3.2 筛选（MVP）

- 时间范围：今天 / 近 7 天 / 自定义起止时间
- trigger 筛选：agent / user / system
- behavior：字符串搜索（MVP），后续可做枚举下拉

#### 7.3.3 行为详情（仅展示数据库字段）

点击列表项打开详情页/抽屉，展示数据库原始字段：

- behavior
- description
- timestamp
- trigger
- duration_minutes
- parameters（数组，逐条展示 value / quantity / reason / extra，extra 以 JSON 展示）

#### 7.3.4 轻管理（Care）

轻管理入口放在“动态”页，面向 C 端用户的“照顾悠酱”操作，避免后台感：

- MVP：零花钱
  - 发放零花钱（增加）
  - 设置零花钱（设定为某个值）
- 可选（后续迭代）：一键补体力、清空今日已执行行为

### 7.4 对话（Chat）

#### 7.4.1 user_name 设置

- user_name 由用户在“设置（Settings）”页面设置
- 首次进入对话页若未设置，提示用户先设置
- MVP 不强制做本地持久化（UI 稿阶段不写入浏览器存储）；后续如需要可再加浏览器存储

#### 7.4.2 对话能力（MVP）

- 消息流（用户/悠酱）
- 输入框发送
- 发送中状态与失败提示
- 可选：清空会话（对 C 端用户更易理解）

## 8. 数据与接口（需求级定义）

说明：本节只定义 Web 页面所需数据能力，不包含具体工程实现细节。

### 8.1 状态

- 获取角色状态：GET /api/nodejs/state/character
- 获取世界状态：GET /api/nodejs/state/world

### 8.2 动态（行为/轻管理）

- 行为查询：GET /api/nodejs/behavior?from=&to=&trigger=&behavior=&page=&page_size=
- 行为详情：GET /api/nodejs/behavior/:id
- 发放/设置零花钱：POST /api/nodejs/state/allowance

### 8.3 对话

- 发送消息：POST /api/nodejs/chat
  - 入参：{ user_name, message }
  - 返回：{ reply, metadata? }

## 9. 非功能需求

- 移动端适配：响应式布局（折叠菜单），触控友好（点击区域 >= 44px）
- 性能：
  - 动态列表支持分页/按天加载
- 可用性：
  - 状态不轮询，用户通过刷新按钮主动更新
  - 网络失败提示清晰
  - 不阻塞主流程（例如状态接口失败不影响对话页打开）
- 安全：
  - 前端不暴露任何密钥
  - 不在日志中打印敏感信息

## 10. 里程碑（MVP 交付范围）

- Dashboard：状态卡片 + 刷新按钮 + 一句话摘要
- Map：示意地图（北-中-南布局）+ 当前地点高亮 + 地点简介
- Activity：行为时间线 + 筛选 + 详情（仅库字段）+ 零花钱入口（发放/设置）
- Chat：可配置 user_name + 消息流 + 发送 + 失败提示
