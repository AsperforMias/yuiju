# 需求

## 背景

去掉 Graphiti 后，长期记忆先不做事件检索型知识库，而是先聚焦“人物记忆”。  
目标是让悠酱稳定记住每个朋友，并在互动时直接读取这个人的完整档案。

## 方案

长期记忆只保留 `people`：

- 每个朋友对应一个独立的 md 文件。
- 需要读取某个朋友时，直接加载整份人物文件，不做向量检索。
- 人物文件是当前阶段的人物长期记忆真相源。

## 目录结构

```text
memory/
  people/
    kobayashi.md
    mika.md
```

## 人物文件内容

每个人物文件应尽量半结构化，至少包含：

- 基本信息：身份、称呼、常见地点、初识方式
- 关系状态：关系类型、亲密度、信任度、最近趋势
- 喜好与厌恶：喜欢的话题、讨厌的事、互动偏好
- 近况：最近在忙什么、近期情绪、近期目标或困难
- 共同经历摘要：重要互动和共同回忆
- 当前待跟进：约定、承诺、下次见面可继续的话题

## 使用方式

- 当当前互动对象明确时，直接读取对应人物文件全文。
- 人物文件用于给悠酱提供稳定的人物认知，不需要再从历史碎片中检索。
- 当前阶段先保证人物页可持续更新、可阅读、可手动修正。

## 第一阶段范围

- 只做 `people`，不单独设计 `event` 长期记忆。
- 先解决“记住人”和“按人读取”。
- 后续如果人物数量变多，再考虑补充更复杂的索引或检索能力。
