# 日记迭代

## 目标

为了提升拟人效果，记忆查询主入口改为 `Episode + Diary + Graphiti`。

## 职责划分

- `Episode`：事件真相源，同时负责“今天”的实时记忆查询
- `Diary`：过去经历的主要回忆入口
- `Graphiti`：长期稳定事实入口

## 查询路由

记忆查询路由保留以下三类来源：

- `Episode`：只允许查询今天的内容
- `Diary`
- `Graphiti`

路由规则：

- 今天发生了什么：查 `Episode`
- 昨天及更早的经历回忆：查 `Diary`
- 长期偏好、关系、长期计划：查 `Graphiti`

以下来源不再进入主记忆查询路由：

- `Plan State`
- 其他临时读模型

## 当前结论

- `Episode` 继续保留，但只负责今天的实时记忆查询
- `Episode` 是 `Diary` 和 `Graphiti` 的上游数据源
- `Diary` 负责昨天及更早的“经历回忆”
- `Graphiti` 负责“长期事实”

## 后续方向

后续继续设计：

- `Diary` 的数据结构
- `Diary` 的生成时机
- `Diary + Graphiti` 的查询路由策略
